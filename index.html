Telegram Personal Assistant Bot

LLM-Powered Personal Management System

專案目標

開發一個基於 Telegram 的智能個人助理機器人

核心理念

傳統的個人管理工具需要用戶學習複雜的指令或操作介面。 本專案利用 LLM 的強大語言理解能力， 讓用戶以最自然的方式與助理對話，就像與真人助理交流一樣。

現有解決方案的局限性

記帳應用程式

  • 需要手動選擇類別
  • 介面複雜，記錄流程繁瑣
  • 類別固定，無法適應個人習慣
  • 需要多次點擊才能完成一筆記錄

待辦事項應用程式

  • 需要填寫表單
  • 日期時間選擇器操作麻煩
  • 無法理解自然語言表達
  • 優先級需手動設定

提醒應用程式

  • 設定重複提醒的介面複雜
  • 無法處理複雜的時間表達
  • 多個時間點需分別設定

筆記應用程式

  • 需要在多個 App 之間切換
  • 分類整理耗時
  • 搜尋功能受限於關鍵字匹配

使用者痛點

我只是想快速記錄「午餐 150」，為什麼要：打開 App → 選擇類別 → 輸入金額 → 選擇日期 → 確認？

筆記、備忘錄沒時間整理，需要的時候花了很多時間在不同層級的目錄中尋找

為什麼需要 LLM？

傳統方式

用戶: "早餐 80"
系統: 無法理解 // 需要固定格式
需要: "/expense 80 food"

LLM 方式

用戶: "早餐 80"
LLM: 自動理解 ✓
{
  "amount": 80,
  "category": "飲食"
}

LLM 能夠解決的問題

語義理解

  • 理解模糊、口語化的表達
  • 處理多樣的時間表達方式
  • 提取關鍵資訊（金額、日期、類別）

上下文感知

  • 記住用戶習慣
  • 理解代詞和省略
  • 處理多輪對話

智能分類

  • 自動判斷支出類別
  • 學習用戶的分類習慣
  • 動態適應，不寫死規則

參數提取

  • 從自然語言中提取結構化資料
  • 處理不完整資訊並合理補全
  • 識別意圖並路由到正確功能

場景 1：智能記帳

今天中午跟同事吃飯花了大概 200 多
已記錄：午餐 $200 (飲食 - 午餐)
Netflix 又扣款了 390
已記錄：Netflix 月費 $390 (訂閱 - 串流影音)
// LLM 自動提取
{
  "intent": "create_expense",
  "amount": 390,
  "category": "訂閱",
  "subcategory": "串流影音"
}

場景 2：時間理解

明天下午 3 點要開會
已新增：開會
時間：2025-11-02 15:00
每週一早上 9 點提醒我開週會
已設定重複提醒：開週會
每週一 09:00
30 分鐘後提醒我
好的，將在 15:30 提醒你
